## 一、整体布局

### 1.1 框架结构（base.html）

所有页面共用同一套布局框架，分为三个区域：

```
┌──────────────────────────────────────────┐
│ 导航栏 navbar（bg-primary）               │
│  📒 记账本  |  流水  |  统计  |  分类     │
├──────────────────────────────────────────┤
│ 消息提示区域（messages）                   │
│  (成功/错误/警告 提示条，可关闭)            │
├──────────────────────────────────────────┤
│ 内容区 content（各页面独有）                │
│                                          │
│                                          │
└──────────────────────────────────────────┘
```

| 区域 | 说明 |
|------|------|
| 导航栏 | 品牌名"📒 记账本"居左，导航链接（流水、统计、分类）居左 |
| 消息提示 | Django messages 渲染为 Bootstrap alert，支持自动关闭 |
| 内容区 | 各页面自定义内容，固定宽度 container |

### 1.2 公共交互

- 所有表单的输入框统一使用 Bootstrap `form-control` 样式
- 表单验证失败时，字段下方显示红色错误提示文字
- 删除操作需要二次确认（`confirm()` 弹窗）
- 页面底部加载 Bootstrap JS bundle，各页面可追加自定义 JS

## 二、流水列表页

> 路径：`/` | 模板：`transaction_list.html`

### 2.1 页面布局

```
┌──────────────────────────────────────────────┐
│ 流水记录                        [+ 新增流水]  │ ← 标题行 + 操作按钮
├──────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│ │   收入   │ │   支出   │ │   结余   │      │ ← 月度统计卡片区
│ │ +xxx.xx │ │ -xxx.xx │ │ xxx.xx  │      │
│ └──────────┘ └──────────┘ └──────────┘      │
├──────────────────────────────────────────────┤
│ [年▼]  [月▼]  [类型▼]  [分类▼]  [筛选]    │ ← 筛选条件栏
├──────────────────────────────────────────────┤
│ 日期    │ 类型 │ 分类  │ 金额   │ 备注 │ 操作 │ ← 流水表格
│ 06/29  │ 收入  │ 💰工资 │ +xxx │ xxx  │ 编辑 删除 │
│ 06/28  │ 支出  │ 🍜餐饮 │ -xxx │ xxx  │ 编辑 删除 │
│ ...                                                │
└──────────────────────────────────────────────┘
```

### 2.2 功能区域说明

| 区域 | 元素 | 说明 |
|------|------|------|
| 标题行 | 标题文字 + 新增按钮 | 按钮跳转 `/create/` |
| 统计卡片 | 3 张 card 横向等宽排列 | 绿色=收入，红色=支出，蓝色=结余 |
| 筛选栏 | 4 个下拉框 + 筛选按钮 | 年/月/类型/分类，GET 方式提交 |
| 流水表格 | table-hover 响应式表格 | 空数据时显示"暂无流水记录"占位 |
| 操作列 | 编辑链接 + 删除表单 | 编辑跳转编辑页；删除内联表单+确认弹窗 |

### 2.3 交互说明

- 筛选条件变更后点击"筛选"刷新页面（GET 请求）
- 删除需弹窗确认，确认后 POST 提交
- 月度统计随筛选年月变化

## 三、新增/编辑流水页

> 路径：`/create/`（新增）、`/{id}/edit/`（编辑）
> 模板：`transaction_form.html`

### 3.1 页面布局

```
┌──────────────────────────────────────┐
│  ┌──────────────────────────────┐   │
│  │  新增流水 / 编辑流水          │   │ ← card-header
│  ├──────────────────────────────┤   │
│  │  类型：  [▼ 收入/支出]       │   │
│  │  分类：  [▼ 分类下拉]        │   │ ← 表单区域
│  │  金额：  [ 输入框 ]          │   │    居中布局
│  │  日期：  [ 输入框 ]          │   │    col-md-6
│  │  备注：  [ 输入框 ]          │   │
│  ├──────────────────────────────┤   │
│  │  [← 返回]         [添加/保存] │   │ ← 按钮行
│  └──────────────────────────────┘   │
└──────────────────────────────────────┘
```

### 3.2 功能区域说明

| 区域 | 元素 | 说明 |
|------|------|------|
| 卡片头部 | 标题 | 根据 `is_edit` 显示"新增流水"或"编辑流水" |
| 表单字段 | 类型/分类/金额/日期/备注 | 使用 Django Form 自动渲染 |
| 操作按钮 | 返回 + 提交 | 返回跳转流水列表；提交根据 is_edit 显示不同文案 |

### 3.3 交互说明

- **动态分类加载**：切换"类型"下拉时，通过 AJAX（`/load-categories/?type=X`）动态更新"分类"下拉选项
- 表单验证失败时，字段下方显示红色错误提示
- 编辑页面回填已有数据

## 八、设计说明

### 8.1 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 导航栏背景 | 蓝色 | bg-primary |
| 收入相关 | 绿色 | success / #198754 |
| 支出相关 | 红色 | danger / #dc3545 |
| 结余/汇总 | 蓝色 | primary / #0d6efd |
| 表格表头 | 浅灰 | table-light |
| 页面背景 | 白色 | 默认 |

### 8.2 响应式设计

- 使用 Bootstrap 栅格系统（col-md-*）实现响应式布局
- 大屏：多列并排（统计卡片3列、收入支出2列）
- 小屏：自动堆叠为单列
- 表格使用 table-responsive 防止溢出

### 8.3 用户体验要点

- 所有操作（新增、编辑、删除）后均有成功/失败消息提示
- 删除操作均需二次确认，防止误操作
- 空数据状态有占位提示，不显示空表格
- 导航栏高亮当前页面（可通过 URL 判断）
- 表单提交失败时保留已填写内容，不丢失数据
